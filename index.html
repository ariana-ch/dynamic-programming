Chapter 1

What dynamic programming actually is

Start with the right mental model before memorising recurrences.

Foundation
Definition. Dynamic programming is a method for solving a problem by breaking it into overlapping subproblems, solving each subproblem once, and reusing those answers.

The two phrases that matter are overlapping subproblems and reusable state.

Intuition. DP is not “fill in a table.” The table is only storage. The hard part is choosing a state that contains exactly the information needed to continue correctly.
Example objective: Compute the nth Fibonacci number while contrasting naive recursion with memoized recursion so you can see exactly where repeated work is removed.
def fib_bad(n):
    if n <= 1:
        return n
    return fib_bad(n - 1) + fib_bad(n - 2)

memo = {}

def fib(n):
    if n <= 1:
        return n
    if n in memo:
        return memo[n]
    memo[n] = fib(n - 1) + fib(n - 2)
    return memo[n]
Chapter 2

The DP design loop

Learn a repeatable process for building DP solutions.

Core workflow
The design loop.

Step 1. Define the state.
Step 2. State its meaning in one sentence.
Step 3. Write the transition.
Step 4. Set base cases.
Step 5. Choose memoization or tabulation.
Step 6. Identify the final answer state.
Step 7. Check time and memory complexity.
Example objective: Count how many distinct ways there are to reach step n if you may move 1 or 2 steps at a time. This example exists to show how a clean state and recurrence emerge from a simple counting objective.
def count_ways(n):
    if n == 0:
        return 1
    if n == 1:
        return 1
    dp = [0] * (n + 1)
    dp[0] = 1
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
If you cannot finish the sentence “dp[i][j] means ...”, your recurrence is probably not ready.
Chapter 3

Memoization and tabulation

Understand the two main implementation styles and when to use each.

Implementation mindset
Memoization. Top-down recursion plus caching.
Tabulation. Bottom-up evaluation in an order where dependencies are already known.
Example objective: Count how many combinations of coin denominations sum to a target, using recursion with caching so that repeated subproblems like “ways to make the same remaining amount from the same index” are solved once.
from functools import lru_cache

def count_ways(coins, target):
    coins = tuple(coins)

    @lru_cache(None)
    def solve(i, remaining):
        if remaining == 0:
            return 1
        if remaining < 0 or i == len(coins):
            return 0
        return solve(i, remaining - coins[i]) + solve(i + 1, remaining)

    return solve(0, target)
Example objective: Solve exactly the same coin-combination counting problem bottom-up with a table, so you can compare the top-down and bottom-up views of the same state.
def count_ways_tab(coins, target):
    dp = [0] * (target + 1)
    dp[0] = 1
    for coin in coins:
        for s in range(coin, target + 1):
            dp[s] += dp[s - coin]
    return dp[target]
Chapter 4

One-dimensional DP

Master the most common DP shape.

Linear states

In one-dimensional DP, the state is indexed by one main progress variable such as position, amount, value, or length.

Example objective: Given a cost at each index, compute the minimum total cost needed to reach the final index when you may move forward by 1 or 2 steps. This example shows 1D DP for optimisation rather than counting.
def min_cost(cost):
    n = len(cost)
    dp = [0] * n
    dp[0] = cost[0]
    for i in range(1, n):
        best_prev = dp[i - 1]
        if i > 1:
            best_prev = min(best_prev, dp[i - 2])
        dp[i] = best_prev + cost[i]
    return dp[-1]
Example objective: Find the length of the longest strictly increasing subsequence of an array using the classical O(n^2) DP recurrence. This example shows how dp[i] can mean “best answer ending exactly at i.”
def lis(nums):
    n = len(nums)
    dp = [1] * n
    for i in range(n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp, default=0)
The state may look one-dimensional while the true dependency needs one more flag or parameter.
Chapter 5

Two-dimensional DP

Track progress in two objects at once.

Matrix DP
Canonical state. dp[i][j] often means the answer for prefixes A[:i] and B[:j].
Example objective: Compute the length of the longest common subsequence shared by two strings or arrays. This is the standard prefix-versus-prefix DP template.
def lcs_length(a, b):
    n, m = len(a), len(b)
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[n][m]
Example objective: Compute the minimum number of insertions, deletions, and substitutions needed to turn one string into another. This shows how the same 2D prefix structure adapts to an optimisation problem with three candidate transitions.
def edit_distance(a, b):
    n, m = len(a), len(b)
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    for i in range(n + 1):
        dp[i][0] = i
    for j in range(m + 1):
        dp[0][j] = j
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
    return dp[n][m]
Chapter 6

Counting DP

Count huge families of objects without double-counting.

Counting
Example objective: Count how many ways there are to move from the top-left corner of a grid to the bottom-right corner using only right and down moves. This is a clean counting-DP example where each state collects ways from predecessor states.
def count_paths(rows, cols):
    dp = [[0] * cols for _ in range(rows)]
    dp[0][0] = 1
    for i in range(rows):
        for j in range(cols):
            if i > 0:
                dp[i][j] += dp[i - 1][j]
            if j > 0:
                dp[i][j] += dp[i][j - 1]
    return dp[-1][-1]
Example objective: Count how many unordered combinations of coin denominations sum to a target value. This example exists to highlight how loop order controls whether you count combinations or permutations.
def coin_combinations(coins, target):
    dp = [0] * (target + 1)
    dp[0] = 1
    for coin in coins:
        for s in range(coin, target + 1):
            dp[s] += dp[s - coin]
    return dp[target]
If order should not matter, the wrong loop order will silently overcount.
Chapter 7

Bitmask and state-compression DP

Handle problems where the state must remember a chosen subset.

State compression
Bitmask DP. Use an integer to represent a subset of items or satisfied requirements.
Example objective: Compute the minimum cost to visit every node exactly once when the state must remember both the current endpoint and the set of already visited nodes. This is the canonical subset-DP pattern behind travelling-style problems.
def tsp_like(dist):
    n = len(dist)
    INF = 10**18
    dp = [[INF] * n for _ in range(1 << n)]
    dp[1][0] = 0
    for mask in range(1 << n):
        for u in range(n):
            if dp[mask][u] == INF:
                continue
            for v in range(n):
                if mask & (1 << v):
                    continue
                nxt = mask | (1 << v)
                dp[nxt][v] = min(dp[nxt][v], dp[mask][u] + dist[u][v])
    return min(dp[(1 << n) - 1])
Example objective: Count how many subsets have total sum exactly equal to a target. This example is deliberately simple so the role of the bitmask itself is easy to see before moving to more advanced compressed-state problems.
def subset_count(nums, target):
    n = len(nums)
    total = 0
    for mask in range(1 << n):
        s = 0
        for i in range(n):
            if mask & (1 << i):
                s += nums[i]
        if s == target:
            total += 1
    return total
State compression is only practical when the number of tracked bits is small enough.
Chapter 8

Interval, partition, and tree DP

Learn the structural DP families that appear once prefix states are not enough.

Structural DP
Example objective: Solve a problem on every subarray [l, r] by trying every split point inside that interval. This is the standard interval-DP skeleton for merge, partition, and game-on-subarray problems.
def interval_dp(arr):
    n = len(arr)
    dp = [[0] * n for _ in range(n)]
    for length in range(2, n + 1):
        for l in range(n - length + 1):
            r = l + length - 1
            best = float('inf')
            for k in range(l, r):
                best = min(best, dp[l][k] + dp[k + 1][r])
            dp[l][r] = best
    return dp[0][n - 1]
Example objective: Compute the size of every subtree in a rooted tree. This is the simplest tree-DP example and exists to make the child-to-parent aggregation pattern explicit before more difficult subtree states.
def subtree_sizes(graph, root=0):
    n = len(graph)
    size = [0] * n

    def dfs(u, parent):
        size[u] = 1
        for v in graph[u]:
            if v == parent:
                continue
            dfs(v, u)
            size[u] += size[v]

    dfs(root, -1)
    return size
Tree-DP lesson. Once the state is defined on subtrees, the main question becomes how child answers combine.
Chapter 9

Hybrid state-space DP

Recognise when the right state should be explored as a graph.

DP + graph view

Sometimes the state is correct, but the best implementation is Dijkstra or BFS over states rather than a plain nested-loop tabulation.

Example objective: Find the shortest path when the state must track both your current node and which requirements or item types have already been collected. This is the key idea behind many (node, mask) shortest-path-plus-DP problems.
import heapq

def shortest_with_mask(graph, start, start_mask, full_mask):
    n = len(graph)
    INF = 10**18
    dist = [[INF] * (full_mask + 1) for _ in range(n)]
    pq = [(0, start, start_mask)]
    dist[start][start_mask] = 0

    while pq:
        d, u, mask = heapq.heappop(pq)
        if d != dist[u][mask]:
            continue
        for v, w, add_mask in graph[u]:
            nxt_mask = mask | add_mask
            nd = d + w
            if nd < dist[v][nxt_mask]:
                dist[v][nxt_mask] = nd
                heapq.heappush(pq, (nd, v, nxt_mask))

    return min(dist[u][full_mask] for u in range(n))

HackerRank pattern library

Map common HackerRank dynamic-programming challenge styles to the right modelling template.

Problem targeting
Pattern map.

Sequence comparison. Use 2D prefix DP.
Reformulate then solve a small 1D cost problem. Often appears disguised as an operations problem.
Independent dimensions plus combinatorial merge. Solve one dimension first and then combine.
Node plus subset state. Use (node, mask) or a similar compressed state.
Example objective: Recover one actual longest common subsequence, not just its length, by backtracking through the completed DP table. This example exists to show how to reconstruct a solution object after computing DP values.
def lcs_string(a, b):
    n, m = len(a), len(b)
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    out = []
    i, j = n, m
    while i > 0 and j > 0:
        if a[i - 1] == b[j - 1]:
            out.append(a[i - 1])
            i -= 1
            j -= 1
        elif dp[i - 1][j] >= dp[i][j - 1]:
            i -= 1
        else:
            j -= 1
    return ''.join(reversed(out))
Chapter 12

Optimisation techniques

Bridge the gap between a correct recurrence and one that actually passes.

Advanced practice
Space compression. Keep only the rows or layers that are still needed.
Precomputation. Prefix sums, combinations, valid-mask lists, and helper arrays can make slow transitions feasible.
Transition pruning. Reduce candidate predecessors when structure allows it.
Example objective: Compute LCS length while reducing memory from a full 2D table to one row at a time. This example exists to show that optimisation changes storage strategy without changing the mathematical state meaning.
def lcs_length_compressed(a, b):
    n, m = len(a), len(b)
    prev = [0] * (m + 1)
    for i in range(1, n + 1):
        cur = [0] * (m + 1)
        for j in range(1, m + 1):
            if a[i - 1] == b[j - 1]:
                cur[j] = prev[j - 1] + 1
            else:
                cur[j] = max(prev[j], cur[j - 1])
        prev = cur
    return prev[m]
Do not compress memory until you are sure which older values are still required. Compression is a correctness change, not just a cosmetic change.
